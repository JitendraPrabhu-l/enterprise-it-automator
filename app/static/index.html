Submit a ticket → agent plans via MCP tools → sensitive actions pause for human approval

How this system works

  1. Tools & framework: a LangGraph agent plans against a real MCP (Model Context Protocol) server — 3 domain servers (identity, access, ticketing), 9 tools total, discovered live via tools/list on every plan, not hardcoded.
  2. HITL (human-in-the-loop): sensitive actions (disable/enable a user, revoke access) pause the agent via LangGraph's interrupt() and require a real person's sign-off before they execute — never auto-approved.
  3. Approval: a reviewer holding a valid token decides via the dashboard, or via Telegram (if configured) — both paths share the exact same authorization check.
  4. Rejection: a rejected approval marks the ticket rejected immediately and the agent never runs the sensitive step.
  5. Escalation: a pending approval left undecided past its SLA deadline is auto-flagged escalated by a background sweep, so nothing silently sits forever.
  6. AG-UI: "Submit & Watch Live" streams the agent's plan/execute/tool-call events over the AG-UI protocol in real time, instead of waiting for the whole run to finish.
  7. Failure handling: a step that fails on a stale assumption (e.g. already disabled) triggers a replan instead of a hard error; a ticket whose action never actually ran is marked failed, not a false "completed."

System flow

A step that succeeds after a replan (or on the first try) marks the ticket completed; one whose required action never actually ran (e.g. re-offboarding an already-disabled employee) is marked failed instead of a misleading "completed."
